I was in need of creating some simple flowcharts, and figured I'd find a basic program online that would suffice. Easy enough - right? Not so. It seems a simple search for flowcharts spawned all kinds of other charting and diagramming programs providing a lot more than the simple flowcharts I wanted to create. In addition to the desktop apps I found, were a couple web-based SaaS apps, one of which, Lucid Chart, already works with Google Apps and Google Drive. Definitely a benefit if you wanted to collaborate, but I didn’t need that. Furthermore, it wasn't clear when installing the Google Apps version of Lucid Chart through the marketplace how to use the free version after the trial would have expired.

Knowing there had to be some open source options, I continued searching and found 2 viable options with different approaches: Dia, and GraphViz.

Dia

Dia is pretty much just what you'd expect in a flowchart program – a graphical way to place shapes onto a surface with a straightforward way of connecting them. The line tool creates directed edges, firmly connecting shapes. Moving shapes around maintains the connections between them, unlike a surprising number of other flowcharting programs I've used before.

Dia (https://wiki.gnome.org/action/show/Apps/Dia) is GTK based, so requires the X Windows emulation system to be installed. As for Mac OS X 10.8 (Mountain Lion), X11 is no longer included as part of the standard development bundle, so requires installation of XQuarts. XQuartz comes as a DMG, and installing it was pretty straightforward. After installing it, you'll be able to run the Dia.app package.

GraphViz

GraphViz (http://www.graphviz.org/) is quite different from Dia. I expected a similar interface for creating flowcharts, so was a bit confused at first by what I found on their website. GraphViz is essentially a command-line program for creating graphs in a number of different formats. It utilizes the DOT language and being a coder, I was quite accustomed to using text editors and was intrigued by this approach.
